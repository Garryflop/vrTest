# Blockchain VR Simulation - Architectural Documentation

## 1. Core Architecture (Автозагрузки)
*   **`NetworkManager.gd`**
    *   Глобальный маршрутизатор (синглтон).
    *   **Данные**: Хранит словарь `graph = { node_id: [соседние node_id] }` и `nodes = { node_id: ссылка на объект }`.
    *   **Ключевая логика**: Метод `find_path(start, end)` реализует алгоритм Поиска в Ширину (BFS). Он обходит граф пакетами и строит путь *только* через те ноды, у которых параметр `is_on == true`.
*   **`SignalBus.gd`** (`Signals`)
    *   Глобальная шина сигналов. Через нее происходит общение не связанных напрямую узлов (например, проигрывание звука, старт/стоп логики).

## 2. Физические блоки-узлы (Ноды графа)
*   **`blockchain_component.gd`**
    *   Основной скрипт (визуального и логического) блока.
    *   Управляет подключениями: генерирует входные и выходные точки (snap zones/anchors) с помощью функций `setup_connectors()`.
    *   **Взаимодействия (XR Tools)**:
        *   Клик по блоку обрабатывается встроенной функцией `pointer_event(event)`, улавливающей нажатие курка (`event_type == 2`). 
        *   Вызов переходит в `_on_toggle_state()`, меняя внутреннюю переменную состояния (красный/зеленый) и напрямую влияя на логику пути в `NetworkManager`.
    *   **Сброс**: Метод `reset()` обнуляет физические импульсы `linear_velocity / angular_velocity` и возвращает куб в сохраненную `initial_transform`.

## 3. Маршрутизация Данных (Визуализация пути)
*   **`data_packet.gd`** (Желтый кубик-пакет)
    *   **Отправка**: Метод `start_journey(path_nodes)` принимает готовый массив из `NetworkManager`.
    *   **Движение**: Используется Godot 4 `Tween`. Пакет плавно перелетает (`tween_property`) от текущей позиции к позиции `global_transform.origin` следующего узла из списка.
    *   **Окончание**: Достигнув финального узла, пакет коротко пульсирует (изменяет `scale` вверх-вниз) и вызывает `queue_free()`.
*   **`tether_line_cylinder.gd` / pipes**
    *   **Функция**: Отрисовка физической «трубы» между двумя узлами графа.
    *   **Логика**: В `update_cylinder()` вычисляется расстояние (для `scale.y`) и угол (через `look_at()`) между узлами. Чтобы труба соединяла центры кубов, ей добавляется поворот в `PI / 2` радиан, так как базовая ось цилиндров смотрит вверх.

## 4. VR Управление и Пользовательский Интерфейс
*   **`player.tscn`** 
    *   Стандартный префаб Godot XR Tools. Оснащен компонентами `FunctionPointer` (Лазер) и `FunctionPickup` (Захват).
    *   Лазеры настроены на: `show_laser = 1` (скрываться вхолостую) и `collision_mask = 5242887` (Смешение системного слоя 1, слоев кубов 2 и 3, а также слоев интерфейса 21).
*   **`main_board.gd` / `board_button.gd`**
    *   **Функция**: 3D-Панель управления (Отправить децентрализованно, Отправить централизованно, Рестарт).
    *   **Взаимодействие**: Кнопки наследуют `StaticBody3D` (настраиваемый под XR Tools аналог коллайдера). Скрипт-компонент кнопок имеет метод `pointer_event`, который пересылает клики (через сигнал `action_triggered`) на главную доску. Главная плата, в свою очередь, передает команды в основной скрипт уровня `level_1_test.gd`.

## 5. Игровой Мир (`level_1_test.gd`)
*   Оркестратор сцены. Отвечает за:
    1. Инициализацию графа (`NetworkManager.clear_graph()`).
    2. Очистку старых путей, если запрошен `reset`.
    3. Асинхронные бесконечные циклы пакетов: функции `send_routine(...)` вызывают `send_data()`, ждут несколько секунд (`await get_tree().create_timer(3.0).timeout`), и если цикл (например `is_decent_looping`) все еще активен — отправляют пакет снова.
